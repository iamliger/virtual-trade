## 3. 🚨 초보자가 반드시 겪는 라이브러리 인식 에러 해결법

  ```json
  {
      "python.defaultInterpreterPath": ".\\venv\\Scripts\\python.exe"
  }
  ```

## 4. 🌐 내 코드를 인터넷(GitHub)에 안전하게 백업하는 방법

개발이 한 단계 끝날 때마다 내 코드를 인터넷 저장소(`origin main`)에 저장하는 공식 3단계 규칙입니다. 

### ① 실전 터미널 명령어 세트
```bash
# 1. 내가 새로 만들거나 수정한 모든 파일을 백업 바구니에 담습니다. (점 '.' 필수)
git add .

# 2. 이번 저장에 어떤 내용을 바꿨는지 한글로 메모(이름표)를 남깁니다.
git commit -m "작업 내용 요약 입력 (예: 메인 엔진 조립 완료)"

# 3. 인터넷 저장소(GitHub)의 메인 공간(main)으로 코드를 쏘아 올립니다.
git push origin main
```

#### Q. git push를 쳤는데 "[rejected] main -> main (fetch first)" 에러가 발생하며 업로드가 안 돼요!
- **해결 방법**:  
  1. 인터넷 파일을 내 컴퓨터로 강제 병합합니다: `git pull origin main --allow-unrelated-histories`
  2. 만약 터미널이 멈춘 것처럼 검은 창이 뜨면 `Esc` -> `:wq` -> `Enter`를 쳐서 빠져나옵니다.
  3. 다시 `git push origin main`을 실행하면 시원하게 업로드됩니다.

## 11. 🛠️ 트러블슈팅 (자주 발생하는 에러)

### ① KeyError: 'title'
- **현상**: 뉴스 데이터를 가져오던 중 'title' 키를 찾지 못해 프로그램이 멈춤.

## 20. 🏗️ GUI-로직 통합 시 주의사항
- **변수 정의(Undefined Variable)**: 함수 선언부와 호출부의 매개변수 이름이 일치해야 하며, 특히 스레드 간 데이터 이동 시 변수 생명 주기를 주의해야 함.
- **스레드 안전(Thread-Safe)**: `threading`을 사용해 백엔드 로직을 돌릴 경우, 화면 갱신은 반드시 메인 스레드(`self.after`)를 통해 수행하여 런타임 충돌 방지.

- **콤보박스 변경 즉시 반응**: 종목 변경 시 하단 로그에 즉시 기록되며, 리포트 창이 초기화되어 이전 종목과의 데이터 꼬임을 방지함.
