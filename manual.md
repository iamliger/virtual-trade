## 2. 오랜만에 컴퓨터를 켰을 때 꼭 해야 하는 필수 명령어 (Git Bash 터미널)

컴퓨터를 새로 켜거나 VS Code를 다시 열면, 반드시 아래 2개의 명령어를 터미널에 쳐서 **가상환경**을 깨워야 합니다.

```bash
# ① 프로젝트 폴더로 정확하게 이동하기 (윈도우 D드라이브 기준)
cd /d/ProjectWorkspace/AI/virtual-trade

# ② 내 방 전용 파이썬 가상환경(venv) 활성화하기
source venv/Scripts/activate
```
* 💡 **매우 중요 (눈으로 확인하기)**: 위 명령어를 치고 났을 때, 터미널 줄 맨 앞에 **`(venv)`** 라는 글자가 초록색으로 나타나야 준비가 끝난 것입니다. 이게 없으면 프로그램이 작동하지 않고 에러가 납니다.

## 3. 🚨 초보자가 반드시 겪는 라이브러리 인식 에러 해결법

### Q. 코드를 잘 짰는데 `yfinance`나 `ollama` 밑에 빨간 줄(또는 노란 줄)이 뜨며 에러라고 나와요!
- **원인**: 컴퓨터에 기본 설치된 엉뚱한 순정 파이썬을 에디터가 바라보고 있어서 그렇습니다. 우리가 설치한 도구들은 프로젝트 폴더 안의 전용 파이썬(`venv`)에 들어있기 때문에 시선을 돌려주어야 합니다.
- **가장 확실한 해결법 (자동화 설정)**:  
  프로젝트 폴더 바로 아래에 `.vscode` 폴더를 만들고 `settings.json` 파일을 생성한 뒤 아래 내용을 넣어두면, 언제 컴퓨터를 켜든 에디터가 알아서 상대경로 가상환경을 찾아가므로 에러가 원천 봉쇄됩니다.
  ```json
  {
      "python.defaultInterpreterPath": ".\\venv\\Scripts\\python.exe"
  }
  ```

## 4. 🌐 내 코드를 인터넷(GitHub)에 안전하게 백업하는 방법

개발이 한 단계 끝날 때마다 내 코드를 인터넷 저장소(`origin main`)에 저장하는 공식 3단계 규칙입니다. 

### ① 실전 터미널 명령어 세트
```bash
# 1. 내가 새로 만들거나 수정한 모든 파일을 백업 바구니에 담습니다. (점 '.' 필수)
git add .

# 2. 이번 저장에 어떤 내용을 바꿨는지 한글로 메모(이름표)를 남깁니다.
git commit -m "가상환경 구축 및 매뉴얼 작성 완료"

# 3. 인터넷 저장소(GitHub)의 메인 공간(main)으로 코드를 쏘아 올립니다.
git push -u origin main
```

### ② 만약 최초 업로드 시 아이디/비밀번호(로그인) 창이 뜬다면?
GitHub에 처음 코드를 올릴 때는 본인 확인 창이 뜰 수 있습니다. 
1. 브라우저 로그인 창이 뜨면 **[Sign in with your browser]**를 눌러 로그인해 줍니다.
2. 만약 **Personal Access Token (토큰 발행)** 창이 뜬다면, 깃허브 웹사이트 설정에서 발급받은 비밀번호 대용 토큰 코드를 입력해 주어야 접속 권한이 승인됩니다.

## 6. 🚫 GitHub 백업 시 절대로 올리면 안 되는 파일들 (.gitignore)
프로젝트 폴더 안의 `venv`(가상환경 폴더)는 용량이 수백 MB로 거대하며, 내 컴퓨터 안에서만 작동하는 파일들이 가득합니다. 이를 그대로 GitHub에 올리면 저장소가 무거워지고 업로드 에러가 발생합니다.

### ⚙️ 해결 및 예방 방법
1. 프로젝트 루트에 `.gitignore` 파일을 생성하고 제외할 폴더 이름(`venv/`)을 적어둡니다. 이렇게 하면 Git이 해당 폴더를 자동으로 무시합니다.
2. 대신, 내가 설치한 라이브러리 목록만 가볍게 텍스트로 뽑아서 공유합니다.
```bash
# 라이브러리 목록 추출 명령어
pip freeze > requirements.txt

# 다른 컴퓨터에서 목록을 보고 똑같이 한 번에 설치하는 명령어
pip install -r requirements.txt
```
이 방식을 사용해야 전 세계 개발자들과 표준 규격으로 깔끔하게 프로젝트를 공유할 수 있습니다.
