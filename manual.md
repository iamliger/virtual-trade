
## 4. 🌐 내 코드를 인터넷(GitHub)에 안전하게 백업하는 방법

개발이 한 단계 끝날 때마다 내 코드를 인터넷 저장소(`origin main`)에 저장하는 공식 3단계 규칙입니다. 

### ① 실전 터미널 명령어 세트
```bash
# 1. 내가 새로 만들거나 수정한 모든 파일을 백업 바구니에 담습니다. (점 '.' 필수)
git add .

# 2. 이번 저장에 어떤 내용을 바꿨는지 한글로 메모(이름표)를 남깁니다.
git commit -m "가상환경 구축 및 매뉴얼 작성 완료"

# 3. 인터넷 저장소(GitHub)의 메인 공간(main)으로 코드를 쏘아 올립니다.
git push -u origin main
```

#### Q. git push를 쳤는데 "[rejected] main -> main (fetch first)" 에러가 발생하며 업로드가 안 돼요!
- **상황 이해**: 깃허브(인터넷) 웹사이트에 내가 모르는 파일(예: README.md)이 먼저 만들어져 있어서 발생하는 거절 메시지입니다. 깃허브의 자식과 내 컴퓨터의 자식을 먼저 하나로 합쳐준 뒤에 올려야 합니다.
- **해결 방법**:  
  1. 인터넷 파일을 내 컴퓨터로 강제 병합합니다: `git pull origin main --allow-unrelated-histories`
  2. 만약 터미널이 멈춘 것처럼 검은 창이 뜨면 `Esc` -> `:wq` -> `Enter`를 쳐서 빠져나옵니다.
  3. 다시 `git push -u origin main`을 실행하면 시원하게 업로드됩니다.


### ② 만약 최초 업로드 시 아이디/비밀번호(로그인) 창이 뜬다면?
GitHub에 처음 코드를 올릴 때는 본인 확인 창이 뜰 수 있습니다. 
1. 브라우저 로그인 창이 뜨면 **[Sign in with your browser]**를 눌러 로그인해 줍니다.
2. 만약 **Personal Access Token (토큰 발행)** 창이 뜬다면, 깃허브 웹사이트 설정에서 발급받은 비밀번호 대용 토큰 코드를 입력해 주어야 접속 권한이 승인됩니다.

## 6. 🚫 GitHub 백업 시 절대로 올리면 안 되는 파일들 (.gitignore)
프로젝트 폴더 안의 `venv`(가상환경 폴더)는 용량이 수백 MB로 거대하며, 내 컴퓨터 안에서만 작동하는 파일들이 가득합니다. 이를 그대로 GitHub에 올리면 저장소가 무거워지고 업로드 에러가 발생합니다.

### ⚙️ 해결 및 예방 방법
1. 프로젝트 루트에 `.gitignore` 파일을 생성하고 제외할 폴더 이름(`venv/`)을 적어둡니다. 이렇게 하면 Git이 해당 폴더를 자동으로 무시합니다.
2. 대신, 내가 설치한 라이브러리 목록만 가볍게 텍스트로 뽑아서 공유합니다.
```bash
# 라이브러리 목록 추출 명령어
pip freeze > requirements.txt

# 다른 컴퓨터에서 목록을 보고 똑같이 한 번에 설치하는 명령어
pip install -r requirements.txt
```
이 방식을 사용해야 전 세계 개발자들과 표준 규격으로 깔끔하게 프로젝트를 공유할 수 있습니다.

## 7. 📖 프로그램 내부 주식 용어 사전 (사전 툴팁 기능)
주식 한자어가 낯선 초보자나 오랜만에 프로그램을 구동하는 관리자가 로그만 보고도 즉각 시스템을 이해할 수 있도록 구현된 기능입니다.

### ① 용어 사전 정의 (`STOCK_DICTIONARY`)
프로그램 백엔드(`trade_manager.py`) 내부에 `Key-Value` 형태의 딕셔너리로 주식 용어를 상시 정의해 둡니다.
- **예수금**: 주식을 사기 위해 내 계좌에 넣어둔 '상태 좋은 현금'
- **매수 / 매도**: 주식을 '사고' '파는' 행위
- **평단가**: 주식을 나누어 샀을 때 계산되는 주당 평균 매입 원가
- **익절 / 손절**: 이득을 보고 파는 것 / 손해를 감수하고 탈출하는 것
- **거래세**: 주식을 팔 때 국가에 내는 세금 (단타의 주된 지출 원인)

### ② 구동 원리
`execute_scalping_buy()` 또는 `execute_scalping_sell()` 함수가 실행되면, 연산 결과 화면 바로 아래에 `💡 [용어 설명]`이라는 머리말과 함께 해당 단어의 정의가 실시간 출력되어 가독성을 높입니다.

## 8. 🧠 로컬 AI (Ollama - Llama3) 판단 엔진 연동 기술
외부 유료 API 비용을 단 1원도 쓰지 않고 내 컴퓨터의 64GB 자원을 100% 활용하여 인공지능 투자 브레인을 탑재한 기능입니다.

### ① 구동 원리 및 데이터 포맷 제어
- `ai_brain.py` 모듈이 작동하면 `yfinance`에서 추출한 '실시간 현재가'와 '최근 5분간의 주가 흐름 배열'을 수집합니다.
- AI가 주식 거래 프로그램이 이해할 수 있는 순수한 데이터 구조만 뱉도록 규격화된 **JSON 형식**(`{ "decision": "결정", "reason": "이유" }`)으로 응답을 한정 지었습니다.

### ② 🚨 중요: 한글 강제 제어 (프롬프트 엔지니어링)
Llama3 순정 모델은 영어를 기본으로 구사하기 때문에, 주식 한자어나 한국어 대답 조건에 압박을 가하는 특수 규칙(`**CRITICAL RULE: The value of 'reason' MUST BE WRITTEN IN KOREAN LANGUAGE ONLY.**`)을 시스템 명령어(System Instruction)에 주입하여 완벽한 한국어 보고서를 받아내도록 가스라이팅을 성공시켰습니다.
